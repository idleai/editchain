## Architecture

### New crates (3)

```
editchain-query    request/response types, RRF fusion, graph algorithms
editchain-index    Tantivy BM25, flat f16 vector index, chunker, snapshots
editchain-embed    Embedder trait, Qwen3 model manifests
```

### Existing crates modified (3)

```
editchain-codec    EC03 framed format with CRC32C checksums
editchain-import   ID collision fix, blob spill, source lanes, streaming reader
editchain-node     CLI (search/retrieve/summarize/tail), daemon module
```

### Key design decisions

1. **Tantivy in RamDirectory** — no fsync overhead; rebuild from chain on restart.
2. **Exact flat f16 vector index first** — HNSW only after measured latency requires it.
3. **f16 vectors** — half precision halves memory vs f32 with negligible accuracy loss.
4. **RRF fusion** — avoids mixing incompatible BM25 and cosine scores.
5. **ArcSwap/RwLock snapshots** — lock-free reader access; atomic publication.
6. **Unix socket daemon** — warm process keeps indexes + model ready; CLI stays clean.
7. **Qwen3-Embedding-0.6B at 512d** — good balance of quality and inference cost.

## CLI surface

```bash
# Lifecycle
editchain init [--path .editchain]
editchain import --sessions-dir <dir> --workspace <dir> [--dry-run]
editchain serve --chain .editchain [--watch]   # daemon mode (warm process)

# Search & retrieval
editchain search "<query>" \
  --mode hybrid|lexical|vector \
  --top 20 \
  --kind message,tool,command

editchain retrieve --op <op-id>

# Summarization (extractive only)
editchain summarize \
  --query "why does X happen" \
  --budget 6000 \
  --strategy extractive|timeline

# Tail / read-your-writes
editchain tail --since <generation> [--follow]
```

## Import correctness fixes (commit 1)

### ID collision
Replaced `seq * 1000 + 1` with `SourcePosition` packing (48-bit record ordinal + 16-bit derived ordinal shifted left by 16). Added `IdError::Overflow` for explicit error handling.

### Blob spill
Extended `BlobSink` trait with `put()` returning `BlobRef`. Added `payload_for()` helper and `ContentAddressedBlobSink`. Records >4 KiB are spilled to content-addressed blobs instead of truncated.

### Deterministic source lanes
Added `derive_source_stream()` that hashes workspace + session file path for per-file deterministic lanes. Import uses per-file sequence numbering from cursor state.

### Streaming reader
Replaced full-file `read_to_end` with `BufRead::read_until(b'\n')` streaming. Memory per read is now one line at a time.

### EC03 frame format
Added checksummed frame format with CRC32C header + payload checksums alongside EC02 read compatibility. Includes format detection helper.

## Query plane

### Lexical index (Tantivy BM25)

16-field schema with:
- `body`, `body_code` — prose and code-aware text fields
- `kind`, `role` — structured filtering
- `actor_id`, `session_id`, `node_id`, `boot`, `seq` — scope and lane filtering
- `clock_ms`, `generation` — time and consistency filtering

BooleanQuery combines full-text search with kind/actor/session filters.

### Vector index (flat f16)

Row-major f16 vectors in active + sealed segments. RoaringBitmap filters for kind and session pre-filtering before dot product scan. Memory estimate: ~98 MiB for 100K vectors at 512d f16.

### Hybrid fusion

Reciprocal Rank Fusion with default k=60. Default candidates: BM25=200, vector=200 → fused top_k=20.

### DAG-aware features

- **FrontierMap** — fast visibility checks without full graph traversal
- **CausalCone** — ancestor/descendant expansion around seed hits
- **CausalCorridor** — shortest parent path from evidence to tip (`retrieve --why`)
- **MMR diversity rerank** — graph-aware Maximal Marginal Relevance penalty
- **Occurrence collapse** — group duplicate chunks by text similarity

## Daemon infrastructure

| Component | Purpose |
|---|---|
| `AppendCoordinator` | Monotonic generation counter per chain directory |
| `ProjectionWatermarks` | Tracks log/hydrated/graph/lexical/vector consistency |
| `ProjectorBus` | Distributes commit notifications to registered projectors |
| `QueryPlane` | RwLock-protected QuerySnapshot for atomic updates |
| `DaemonConfig` | Chain dir, socket path, commit interval configuration |

Consistency modes:
- `--consistency lexical` — current graph + BM25; vector lag reported
- `--consistency hybrid` — wait for full coverage
- `--min-generation <gen> --wait <ms>` — read-your-writes

## Embedding model support

```rust
// Embedder trait with query/document prefix support
pub trait Embedder: Send + Sync {
    fn embed(&self, texts: &[String]) -> Result<Vec<Vec<f32>>, EmbedError>;
    fn embed_query(&self, query: &str) -> Result<Vec<f32>, EmbedError>;
    fn embed_document(&self, text: &str) -> Result<Vec<f32>, EmbedError>;
}
```

Pre-configured manifests:
- **Qwen3-Embedding-0.6B**: 512d, 32K context, mean pooling (default)
- **Qwen3-Embedding-4B**: 512d, 32K context (upgrade path)

Any manifest change (model ID, dimensions, pooling, prefixes) forces a vector rebuild via the `identity()` hash.
