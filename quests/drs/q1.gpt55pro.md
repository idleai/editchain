# Claude Code History Import — Short Implementation Plan

**author:** GPT5.5 Pro   
**trascript:** https://chatgpt.com/share/6a51f299-8e98-83e8-a4aa-dbe41a9ad307  
**Target:** `idleai/editchain`  
**Crate:** `crates/editchain-import/`

## Goal

Import every Claude Code session associated with a workspace into **one edit chain**. Preserve the original JSONL as evidence, while also emitting useful `Message`, `Tool`, `Command`, `File`, and `Reflection` operations.

The importer must be deterministic, idempotent, append-friendly, and safe when several agents or humans contribute concurrently.

## Core decisions

1. **Reference implementation**

   Add `delexw/claude-code-trace` as a reference-only submodule at:

   ```text
   externals/claude-code-trace
   commit: aacd7f0c5dd33ac41f7d17e259b22a723e783c37
   ```

   Study its permissive Rust parser, partial-line handling, parent graph, forks/rewinds, and subagent discovery. Do not depend on it as a Rust crate or copy its viewer-specific filtering.

2. **Raw-first import**

   Every complete JSONL line produces one private `ImportOp` containing either the raw bytes or a blob reference plus a hash.

   ```text
   physical JSONL line
       ├── exactly one ImportOp
       └── zero or more normalized ops
   ```

   Unknown and malformed records are still archived. Normalization may improve later without losing the source evidence.

3. **One chain per workspace**

   Main sessions, background sessions, and nested subagents all enter the same workspace chain. Each physical source stream gets its own deterministic edit-chain node, so concurrent sessions are not forced into a fake global order.

4. **Stable identity**

   Derive operation, actor, session, turn, path, and source-node IDs from versioned hashes of stable source data. Never use current time, random values, file modification time, or directory enumeration order.

5. **Preserve occurrences; deduplicate only the logical view**

   Claude Code can copy the same conversation prefix into multiple session files. Keep every physical occurrence auditable, but collapse equivalent logical events using the external UUID plus a normalized payload hash. If the same UUID has different content, report a collision and keep both.

6. **Conservative file history**

   Emit a materialized `FileOp` only when the source proves the resulting bytes. Otherwise record the edit/tool evidence without inventing an `after` state. Store a path-manifest note so a `PathId` can be resolved back to text.

## Minimal normalization mapping

| Claude Code record | Editchain output |
|---|---|
| Human `user` message | `MessageOp`, human actor |
| Assistant text | `MessageOp`, agent actor |
| `tool_use` | `ToolOp::Start` |
| Matching `tool_result` | `ToolOp::Finish` |
| Bash tool | `ToolOp` plus `CommandOp` lifecycle when enough data exists |
| Summary / compact boundary | `ReflectionOp` |
| Full file attachment or proven write result | `FileOp` plus path manifest |
| Hook, denial, or source error | `NoteOp` or `ErrorOp` |
| Metadata, progress noise, unknown type | raw `ImportOp`; normalize only when useful |

Thinking content should remain private and opt-in. Repeated conversation snapshots embedded inside progress records must not be re-imported as new messages.

## Proposed crate shape

Keep the first version small:

```text
crates/editchain-import/src/
├── lib.rs
├── error.rs

└── claude_code/
    ├── mod.rs
    ├── discover.rs
    ├── reader.rs
    ├── envelope.rs
    └── normalize.rs
```

Expose one main API:

```rust
pub fn import_claude_code(
    request: &DiscoveryRequest,
    options: &ImportOptions,
    ops: &mut dyn OpSink,
    blobs: &mut dyn BlobSink,
    cursors: &mut dyn CursorStore,
) -> Result<ImportReport, ImportError>;
```

The sinks allow tests, local storage, and future remote gateways to reuse the same importer.

## Implementation order

### 1. Pin reference and profile the corpus

Add the submodule and a small profiler that reports record types, maximum line size, malformed lines, session IDs, UUID overlap, parent resolution, tool-result matching, and subagent indicators.

### 2. Build lossless raw import

Implement byte-oriented JSONL reading, partial-final-line deferral, raw hashing, inline/blob storage, deterministic IDs, reports, and import cursors.

Re-importing unchanged files must produce only exact duplicates. Appending to a source must emit only new records. Truncation or rewrite must create a new source generation rather than reuse prior IDs.

### 3. Normalize the main conversation

Add loose envelope parsing, session and turn IDs, human/agent actors, user and assistant messages, tool starts/results, Bash commands, parent links, and logical copied-prefix projection.

### 4. Add agent and file relationships

Discover nested `subagents/agent-*.jsonl` files and their `.meta.json` links. Preserve background sessions, forks, rewinds, sidechains, attachments, compact summaries, file observations, and path manifests.

### 5. Integrate durable storage and CLI

Before bulk import, fix two node-layer issues:

- use a safe batch writer that rotates after each encoded page, or explicitly supports concatenated pages;
- serialize local writers with an exclusive chain lock.

Add:

```text
editchain import claude \
  --workspace <repo> \
  --claude-home ~/.claude \
  --chain .editchain/<chain> \
  [--dry-run]
```

Cursor updates must be atomic and occur only after operations and blobs are durably written.

## Required tests

- Every complete source line creates exactly one raw `ImportOp`.
- Unknown and malformed lines do not stop the file.
- Two imports of unchanged input produce no new accepted operations and no quarantines.
- Appending lines preserves every existing operation byte-for-byte.
- Source enumeration and merge order do not change the accepted op set or logical view.
- Copied prefixes remain visible as occurrences but appear once in the logical conversation.
- Same external UUID with different content is reported, never silently merged.
- Tool results link to starts when IDs exist; dangling results remain preserved.
- Background sessions and nested subagents remain separately attributable.
- A crash before cursor update only causes harmless duplicate replay.

## Completion criteria

The quest is complete when all linked seed and q0 Claude Code histories import into one workspace chain; sessions, actors, turns, tools, commands, branches, and subagents remain queryable; every physical source line is recoverable; a second import is a no-op; an appended source adds only new operations; and concurrent replicas can merge without requiring a global sequence allocator.
