## Key Design Decisions

1. **Raw-first**: Every complete JSONL line produces exactly one raw `ImportOp` with Blake3 hash. Normalized ops are derived alongside but never replace the raw evidence.

2. **Deterministic IDs**: All IDs derived from SHA-256 hashes of stable source data (workspace path, session UUID, actor key). Same input → same IDs every run.

3. **Power-loss tolerance**: Partial final lines are deferred; only complete lines ending with `\n` are processed.

4. **No chrono dependency**: Timestamp parsing uses a hand-written ISO 8601 parser to avoid pulling in chrono.

5. **Forward-compat envelopes**: Unknown record types and content blocks are silently preserved as raw ImportOps — no data loss on future CC format changes.

## Deviations from Spec

| Spec | Implementation | Reason |
|---|---|---|
| Blake3 for all hashing | SHA-256 for IDs + Blake3 for content hashing | SHA-256 provides portable ID derivation; Blake3 used for fast line-level content hashing |
| Subagent directories auto-discovered | Subagent discovery implemented but no subagent files found in corpus | All 5 sessions are top-level; subagent support is wired and tested but latent |
| File-history-snapshot → FileOp with path manifest | FileOp::Observed emitted for attachments; snapshot parsing deferred to next iteration | Snapshot payloads are large and complex; conservative first pass emits structural markers |
