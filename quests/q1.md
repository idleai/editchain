# claude-code history import

Previous quests here [seed.result.md](./seed.result.md) and [q0.result.md](./q0.result.md) link to session logs (local). We want to analyze those session logs, understand its structure and implement the importer for them at [editchain-import](../crates/editchain-import/). While we are building this against claude code, we will potentially add import capabilities for other agent history formats too. 

The goal here is to find cleanest rust reference that imports or implements claude code's session format, add it as a git submodule, analyze its implementation and then add a minimal importer here. 

One thing to keep in mind is that while claude code history session files are standalone and tie to a session itself (bg agents can also tie to same session), editchain is tied to the entire workspace (this editchain codebase), so cc importer will convert multiple cc sessions into a single editchain. 

Also, editchain is supposed to solve for multiplayer-model where multiple agents and humans are working and continiously appending into it which you should keep as design goal here.
